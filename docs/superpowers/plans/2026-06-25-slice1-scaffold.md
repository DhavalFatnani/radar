# Slice 1 — Scaffold That Runs — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a Next.js 15 app that boots locally, serves a `/api/v1/health` healthcheck, renders a blank home route, validates its environment, and proves all of this with green tests.

**Architecture:** A single Next.js 15 App Router project under `src/`, with the layered folder skeleton from the locked stack design (services / ai / integrations / db created later, not in this slice). API route handlers live under `src/app/api/v1/*` and stay thin. Environment is parsed and validated once through a zod schema in `src/lib/env.ts`. Tests run on Vitest, split into `tests/unit` and `tests/integration` per the project layout.

**Tech Stack:** Next.js 15 (App Router) · React 19 · TypeScript (strict) · Zod · Vitest · ESLint 8 + `eslint-config-next` · Prettier · Node 22.

**Source spec:** `docs/superpowers/specs/2026-06-25-phase1-stack-design.md` and `Prompt_Playbook.md` Part 3 → Slice 1.

## Global Constraints

These apply to every task. Values copied verbatim from the locked design / project guides.

- **Runtime:** Node 22 LTS (verified: `v22.22.2`).
- **Framework:** Next.js 15, App Router. Pin to the 15 line (`next@15`) — the stack is locked at 15, do not pull 16+.
- **Language:** TypeScript, `strict` on. End-to-end types.
- **API prefix:** all API routes under `/api/v1/`.
- **API error shape (for any future error response):** `{ error: string, code: string }` with the correct HTTP status — never 200 for errors. (Slice 1 has only a success response; this is the convention later slices follow.)
- **Layer direction:** `app/api` (thin: validate → call service → shape response) → `services/*` → `db/*`. No `services` importing from `app`. (No services exist yet in this slice.)
- **Secrets:** never in the repo. All config flows through the validated `src/lib/env.ts`. `.env.local` is git-ignored; `.env.example` is the documented template.
- **Test location override:** the project design doc and `AGENTS.md` mandate a separate `tests/unit` + `tests/integration` tree. This **deliberately overrides** the global `~/.claude/CLAUDE.md` "test next to the file" rule for this repo, because the project layout is more specific. (Flag for operator during plan review if undesired.)
- **Pre-commit:** run `detect_changes()` (GitNexus) before each commit to confirm only expected scope changed, per project `CLAUDE.md`. Note: this slice is greenfield (all new files, no existing code symbols), so `impact` analysis is N/A here.

## Out of scope / deferred (do NOT build in Slice 1)

- Any data models, Drizzle schema, or DB connection/query (→ Slice 2). The healthcheck is **app-only**; it must not query Postgres.
- Any auth / login (→ Slice 3).
- Any real UI, nav, or section screens (→ Slice 4).
- Moving `Phase0_Platform_Specification.md` / `Prompt_Playbook.md` into `docs/` (deferred — it would churn many references in `CLAUDE.md`/`AGENTS.md`; not needed for "scaffold that runs").

- [ ] **Step 3: Implement `src/lib/env.ts`**

```ts
import { z } from "zod";

const envSchema = z.object({
  NODE_ENV: z.enum(["development", "test", "production"]).default("development"),
  // Slice 1: Neon connection string is wired but optional (no DB usage yet).
  // Slice 2 (data layer) tightens this to required.
  DATABASE_URL: z.string().url().optional(),
});

export type Env = z.infer<typeof envSchema>;

export function parseEnv(
  source: NodeJS.ProcessEnv | Record<string, unknown>,
): Env {
  const result = envSchema.safeParse(source);
  if (!result.success) {
    const issues = result.error.issues
      .map((issue) => `${issue.path.join(".")}: ${issue.message}`)
      .join("; ");
    // Note: reports which vars failed and why, without echoing their values.
    throw new Error(`Invalid environment variables: ${issues}`);
  }
  return result.data;
}

export const env = parseEnv(process.env);
```
