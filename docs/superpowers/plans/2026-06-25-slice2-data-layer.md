# Slice 2 — The Data Layer — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement every Phase 0 §4 data model as a Drizzle/Postgres schema with migrations, a seed script, and integration tests that run against a real Neon Postgres — enforcing the two non-negotiable spec constraints (proof principle, approval gate) at the database level.

**Architecture:** One Drizzle schema per domain under `src/db/schema/`, a shared enums module, a `postgres.js` client in `src/db/client.ts`, forward migrations generated by `drizzle-kit`, and a `src/db/seed.ts` that inserts one row per table in FK order. Integration tests connect to a dedicated **test** database (a Neon branch), apply migrations programmatically, and assert create/read plus the mandatory-field and default-status constraints.

**Tech Stack:** Drizzle ORM (`drizzle-orm/postgres-js`) · drizzle-kit · postgres.js · PostgreSQL 16 (Neon) · zod (v3) · Vitest · dotenv (test env loading).

**Source spec:** `Phase0_Platform_Specification.md` §4 (data models), §4.6 (catalogue graph), and the locked `docs/superpowers/specs/2026-06-25-phase1-stack-design.md` (catalogue = nodes+edges in Postgres).

## Global Constraints

Apply to every task. Verbatim values from the spec/design.

- **Proof principle (§4.2, §8):** a `signal_observations` row MUST have non-null `detected_at`, `source`, and `evidence`. Enforce as `NOT NULL` columns.
- **Approval gate (§4.1, §4.3, §8):** `signal_definitions.status` and `mappings.status` are an enum `proposed | approved | retired`, **defaulting to `proposed`**.
- **Postgres at the core (§10, stack design):** catalogue graph is modeled as `catalogue_nodes` + `catalogue_edges` in Postgres — NO separate graph DB.
- **Layer direction:** `app/api` → `services/*` → `db/*`. This slice builds only `db/*`. No services, no API routes, no UI.
- **Tests:** integration tests run against a **real** database (never mocked), in `tests/integration/`. Unit tests (if any pure logic) in `tests/unit/`.
- **TypeScript strict; zod v3** (do not let a stray zod v4 upgrade in).
- **Secrets:** `DATABASE_URL` / `TEST_DATABASE_URL` only via env; never in the repo. `.env.local`/`.env.test` are git-ignored.
- **Staging discipline:** stage only each task's files explicitly — never `git add -A` (an unrelated `mockups/` tree and `.superpowers/` scratch must stay out).
- **Pre-commit:** run `detect_changes()` (GitNexus) before each commit per project `CLAUDE.md`.

## Design Decisions (flagged for operator review)

These resolve genuine gaps/ambiguities in §4. Override any during plan review.

1. **`companies` table added** (operator-approved): satisfies `signal_observations.company_id` and `leads.company_id`. Minimal now (`company_id`, `name`, `description`, `profile` jsonb, `created_at`); entity resolution (Phase 4) enriches it. The §4.5 lead "company object" is represented by the `companies` row the lead FK-references (not a duplicated embedded blob).
2. **Primary keys:** `signal_definitions.signal_id` is **text** (the semantic `SIG-…` code, because `mappings.required_signals` and `signal_observations.signal_id` reference it). All other PKs are generated **uuid** (`defaultRandom()`).
3. **Enums:** `pgEnum` for closed sets (family, detection_method, strength, false_positive_risk, polarity, entity_type, lifecycle status, pipeline_stage, outreach_mode, commission_type, catalogue node type). **`recheck_cadence` is plain `text`** because §4.1 says "weekly, monthly, quarterly, **etc.**" (open-ended).
4. **Structured sub-objects → `jsonb`** (per playbook "structured fields"): vendor `constraints`/`ideal_customer`/`interview_history`/`credibility`, lead `brief`/`contact_block`, `parameters`, `track_record`, `commission_terms`, contact `categories`/`contact_paths`, catalogue `metadata`. **Simple scalar arrays → `text[]`** (sources, pairs_with, geography, capabilities, required_signals, supporting_signals, disqualifiers, evidence).
5. **Computed fields stored as nullable columns now** (`track_record`, `signal_recipe`, `freshness_verdict`, `entity_match_confidence`, `commission_due`, `dedup_key`) — populated by later phases. No business logic this slice.
6. **Leak-defense logs (§4.7)** `disclosure_log`/`introduction_log`/`dispute_record` modeled as `jsonb` columns on `projects` for this slice; may normalize into tables in Phase 5. **`pipeline_stage` lives on `leads`** (per §4.7 "enum on the lead").
7. **Migrations are forward-only (drizzle-kit idiom).** "Reversible/up-and-down" (playbook acceptance) is satisfied by a `db:reset` script (drop schema cascade + re-migrate) proving a clean rebuild, plus Neon test-branch disposability. No hand-written down-migrations.
8. **Deferred (NOT in this slice):** holding pool (§7.7, stubbed — no §4 model), `bundling_mode` toggle (§4.6, a system setting), full vendor version-history tables (kept as `version` int + `interview_history` jsonb).

```ts
import { drizzle } from "drizzle-orm/postgres-js";
import { migrate } from "drizzle-orm/postgres-js/migrator";
import postgres from "postgres";
import { sql } from "drizzle-orm";
import * as schema from "@/db/schema";

const url = process.env.TEST_DATABASE_URL ?? process.env.DIRECT_URL ?? process.env.DATABASE_URL;
if (!url) throw new Error("No TEST_DATABASE_URL/DIRECT_URL/DATABASE_URL set for integration tests");

// `max: 1` = deterministic single connection; `prepare: false` is safe on pooled or direct.
const client = postgres(url, { max: 1, prepare: false });
export const testDb = drizzle(client, { schema });

export async function migrateTestDb() {
  await migrate(testDb, { migrationsFolder: "./src/db/migrations" });
}

// Truncate every table between tests (RESTART IDENTITY + CASCADE for FKs).
export async function truncateAll(tables: string[]) {
  if (tables.length === 0) return;
  const list = tables.map((t) => `"${t}"`).join(", ");
  await testDb.execute(sql.raw(`TRUNCATE TABLE ${list} RESTART IDENTITY CASCADE;`));
}

export async function closeTestDb() {
  await client.end();
}
```

```ts
import {
  signalFamily, detectionMethod, signalStrength, falsePositiveRisk,
  signalPolarity, entityType, lifecycleStatus,

import { companies } from "./companies";

export const signalDefinitions = pgTable("signal_definitions", {
  signalId: text("signal_id").primaryKey(),                 // e.g. SIG-HIRING-OPS-SURGE
  name: text("name").notNull(),
  family: signalFamily("family").notNull(),
  description: text("description"),
  sources: text("sources").array(),
  detectionMethod: detectionMethod("detection_method"),
  triggerRule: text("trigger_rule"),
  parameters: jsonb("parameters"),
  proofCaptured: text("proof_captured"),
  confirmationRule: text("confirmation_rule"),
  recheckCadence: text("recheck_cadence"),                  // open-ended -> text
  strength: signalStrength("strength"),
  falsePositiveRisk: falsePositiveRisk("false_positive_risk"),
  freshnessWindowDays: integer("freshness_window_days"),
  polarity: signalPolarity("polarity"),
  entityType: entityType("entity_type"),
  pairsWith: text("pairs_with").array(),
  geography: text("geography").array(),
  status: lifecycleStatus("status").notNull().default("proposed"),   // APPROVAL GATE
  origin: text("origin"),
  proposedBy: text("proposed_by"),
  dateAdded: date("date_added"),
  lastReviewed: date("last_reviewed"),
  example: text("example"),
  trackRecord: jsonb("track_record"),                       // computed; empty until outcomes
});

export const signalObservations = pgTable("signal_observations", {
  observationId: uuid("observation_id").primaryKey().defaultRandom(),
  signalId: text("signal_id").notNull().references(() => signalDefinitions.signalId),
  companyId: uuid("company_id").notNull().references(() => companies.companyId),

  source: text("source").notNull(),                                          // MANDATORY (proof)
  evidence: text("evidence").array().notNull(),                              // MANDATORY (proof)
```

```ts
import postgres from "postgres";
import { drizzle } from "drizzle-orm/postgres-js";
import { migrate } from "drizzle-orm/postgres-js/migrator";
import { sql } from "drizzle-orm";
import { config } from "dotenv";

config({ path: ".env.local" });

// Reset uses the DIRECT (unpooled) endpoint — DROP SCHEMA / DDL is unreliable over PgBouncer.
const url = process.env.DIRECT_URL ?? process.env.DATABASE_URL;
if (!url) throw new Error("DIRECT_URL or DATABASE_URL is required for db:reset");

const client = postgres(url, { max: 1, prepare: false });
const db = drizzle(client);

await db.execute(sql.raw("DROP SCHEMA public CASCADE; CREATE SCHEMA public;"));
await migrate(db, { migrationsFolder: "./src/db/migrations" });
await client.end();
console.log("Reset complete: public schema dropped and migrations re-applied.");
```
